# デプロイメント状況確認手順

## 現在の状況

ログから確認できる情報：

- **実行中のリビジョン**: `confluence-chat-build-2025-11-18-007`（古いビルド）
- **エラー**: `API key expired. Please renew the API key.`
- **Secret Manager参照**: `"key": "3"`（バージョン3を参照）

## 問題

新しいビルド（`build-2025-11-18-008`）がまだデプロイされていないか、古いリビジョンがまだ実行中です。

## 確認方法

### 1. Firebase Consoleで確認

1. [Firebase Console](https://console.firebase.google.com/)にアクセス
2. プロジェクトを選択
3. App Hosting → Deployments
4. 最新のデプロイメントの状態を確認
5. `build-2025-11-18-008`が完了しているか確認

### 2. Cloud Runで確認

```powershell
# 最新のリビジョンを確認
gcloud run revisions list --service=confluence-chat --project=confluence-copilot-ppjye --region=us-central1 --limit=5
```

### 3. ログで確認

新しいリビジョンがデプロイされている場合、ログに以下のようなメッセージが表示されます：

```
revision_name: "confluence-chat-build-2025-11-18-008"
```

また、Secret Managerの参照が `"key": "4"` になっていることを確認してください。

## 次のステップ

1. **新しいビルドの完了を待つ**: ビルドが完了するまで数分かかる場合があります
2. **デプロイの確認**: 新しいリビジョンがデプロイされているか確認
3. **動作確認**: 新しいリビジョンでエラーが解消されたか確認

## トラブルシューティング

### 問題: 新しいビルドがデプロイされない

**原因**: ビルドが失敗している可能性があります

**解決方法**:
1. Firebase Consoleでビルドログを確認
2. エラーがあれば修正
3. 必要に応じて再デプロイ

### 問題: 新しいビルドがデプロイされたが、まだ古いリビジョンが実行されている

**原因**: Cloud Runのトラフィックが古いリビジョンにルーティングされている可能性があります

**解決方法**:
1. 数分待ってから再度確認
2. 必要に応じて、Firebase Consoleから手動で新しいリビジョンにトラフィックを切り替え
